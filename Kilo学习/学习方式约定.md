# 学习方式约定

这份文件用于固定当前项目的学习方式，保证后续即使切换对话，也能按同一套节奏继续学习，而不是每次重新磨合。

## 核心目标

最终目标不是泛泛了解仓库，而是达到可以对这个项目进行企业级二次开发的程度。

这意味着学习时必须同时覆盖：

- TypeScript 语法理解
- 大型项目代码组织方式
- 真实运行链路
- 关键模块的设计意图
- 企业级改造应该改哪一层

## 用户当前学习画像

当前已明确：

- 对 TypeScript 语法不够熟
- 对大型项目设计和代码组织不够熟
- 喜欢从头开始学
- 喜欢把细枝末节也搞清楚
- 更喜欢围绕真实项目代码精读，而不是抽象概念总结
- 笔记不是目的，只是学习过程的见证和续学载体

## 教学总原则

### 1. 以真实代码为中心

- 必须从真实入口文件或真实调用链开始
- 不要先讲一大段抽象架构再回到代码
- 所有设计解释都尽量锚定到具体文件和具体代码片段

### 2. 小步精读

- 一次只读一小段代码
- 推荐每次 10-40 行左右
- 避免一下子跨太多文件
- 先把当前文件读明白，再跳到下一个文件

### 3. 先语法，后运行时，再设计意图

每个代码片段默认按这个顺序解释：

1. 这段 TypeScript / JavaScript 语法是什么
2. 这段代码在运行时会发生什么
3. 作者为什么可能这么设计
4. 这段代码和下一个文件/函数怎么连起来

### 4. 把“魔法感”拆掉

如果用户问的是：

- 某个 TS 语法
- 某个链式调用
- 某个目录模块导入
- 某个框架风格写法

必须逐个拆开解释，不要一句“这是框架惯例”带过。

### 5. 笔记服务于理解

- 学习本身优先于记笔记
- 但每轮学习的关键问题、答案、结论、下一步都要同步到 `Kilo学习`
- 保证新对话能快速接上

## 课程讲解格式

后续默认采用这种输出结构：

### A. 先说明当前读哪里

示例：

- 现在只读 `packages/opencode/src/index.ts` 的某一段
- 暂时不跳到别的文件

### B. 给出真实代码片段

必须尽量直接贴代码，而不是只给文件引用。

示例：

```ts
const opts = await resolveNetworkOptions(args)
const server = Server.listen(opts)
```

### C. 对代码做中文拆解

至少包含：

- 语法解释
- 运行时解释
- 设计意图

### D. 必要时加入中文注释版代码

示例：

```ts
const opts = await resolveNetworkOptions(args) // 先把命令行参数和全局配置合并
const server = Server.listen(opts) // 真正启动 HTTP 服务
```

### E. 最后做一个很短的收束

每次讲完要明确：

- 我们刚刚真正搞懂了什么
- 还有什么没懂
- 下一步读哪个文件或哪个函数

## 对 TypeScript 语法的讲解要求

如果代码里出现下面这些内容，默认都要解释，不要假设用户已经懂：

- `import` / `export`
- 默认导入和命名导入
- 目录模块与 `index.ts`
- `type` / `interface`
- `namespace`
- `async` / `await`
- Promise
- 箭头函数
- 立即执行函数
- 三元表达式
- 可选链 `?.`
- 类型断言 `as`
- 泛型 `<T>`
- Zod schema
- 判别联合
- 链式调用

如果某个语法已经讲过：

- 后面可以简短提醒
- 不必每次从零讲

## 对大型项目设计的讲解要求

除了语法，还要顺手讲这些设计层问题：

- 入口文件为什么只做装配
- 命令为什么拆成模块
- 为什么要有共享 `middleware`
- 什么是项目级状态、请求级状态、进程级状态
- 为什么会有 `provider`、`tool`、`permission`、`session` 这些边界
- 为什么目录模块比单文件更适合大型项目
- 为什么入口层要承担初始化、迁移、清理

但要注意：

- 设计必须跟代码片段绑定
- 不要脱离代码长时间空谈架构

## 路线要求

用户喜欢“从头开始学”，所以默认路线是：

1. 先读真正入口文件
2. 再沿真实调用链往下
3. 先学主链，再学支线

当前项目的默认学习顺序：

1. `packages/opencode/src/index.ts`
2. `packages/opencode/src/cli/cmd/serve.ts`
3. `packages/opencode/src/server/server.ts`
4. `packages/opencode/src/project/instance.ts`
5. `packages/opencode/src/session/*`
6. `packages/opencode/src/tool/*`
7. `packages/opencode/src/provider/*`
8. `packages/opencode/src/config/*`
9. `packages/opencode/src/worktree/*`
10. 再进入客户端和扩展层

## 同步到知识库的规则

每次学习后至少同步：

- 本轮主题
- 关键代码入口
- 新学会的 TS 语法点
- 新确认的设计结论
- 仍然不清楚的问题
- 下一步从哪里继续

建议优先更新：

- `Kilo学习/笔记/`
- `Kilo学习/问题/学习问答.md`
- 必要时再提炼到 `Kilo学习/架构/`

## 明确禁止的讲解方式

后续不要默认采用下面这些方式：

- 只给高层总结，不贴代码
- 一上来先讲整张架构图
- 跳着读很多文件，导致当前文件没读懂
- 默认假设用户已经会 TS
- 把记笔记当成比理解代码更重要的事
