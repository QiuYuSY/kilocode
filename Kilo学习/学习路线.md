# 学习与接管路线

## 总目标

达到可以进行企业级二次开发的程度，意味着不仅知道代码在哪，还要知道：

- 系统怎么启动
- 请求怎么流动
- 会话和状态怎么保存
- 工具如何注册和执行
- 模型与 Provider 如何接入
- 客户端如何连接 CLI backend
- 哪些地方适合做企业改造，哪些地方最好少动

## 阶段划分

### 阶段 1：建立全景图

目标：知道这个 monorepo 里谁是核心，谁是外壳，谁是共享层。

重点：

- 根工作区与产品关系
- `packages/opencode/` 的核心地位
- VS Code 扩展、Desktop、Web 与 CLI 的关系

### 阶段 2：吃透 CLI 内核

目标：从命令入口进入，理解 server、session、tool、provider、worktree 的连接关系。

重点：

- `src/index.ts`
- `src/cli/cmd/serve.ts`
- `src/server/server.ts`
- `src/session/`
- `src/tool/`
- `src/provider/`
- `src/worktree/`

### 阶段 3：理解客户端边界

目标：知道 VS Code 扩展怎样把 CLI 能力包装成产品能力。

重点：

- CLI backend 启动方式
- HTTP + SSE 通信模式
- Webview 和 extension 的职责边界
- Agent Manager 的多会话与 worktree 隔离

### 阶段 4：识别企业改造点

目标：把“懂项目”转化成“知道改哪里”。

重点：

- 认证鉴权
- 审计与遥测
- 自研模型或私有网关接入
- 组织级配置
- 权限审批与工具限制
- 多租户与工作区隔离

### 阶段 5：实战改造

目标：选一个真实企业需求，从设计到代码落地做完整闭环。

可选主题：

- 接企业内部模型网关
- 加审计日志与操作留痕
- 加审批流或工具白名单
- 加组织级策略配置

## 当前建议顺序

现在先从阶段 2 开始，因为这是企业二开的主战场。
