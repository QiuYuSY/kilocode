# Monorepo 全景

## 一句话理解

这是一个以 `packages/opencode/` 为平台内核的多产品仓库。CLI 提供 AI agent runtime、HTTP server、session、tool、provider 等核心能力，其余产品大多以客户端身份消费这些能力。

## 顶层技术形态

- 包管理与工作区：Bun workspaces
- 多包编排：Turborepo
- 主要语言：TypeScript
- 关键运行形态：CLI、HTTP server、SSE、SolidJS、VS Code extension

## 产品关系

### 核心平台

- `packages/opencode/`
  - CLI 内核
  - AI agent runtime
  - Hono HTTP server
  - Session 管理
  - Tool 执行
  - Provider / Model 集成

### 核心客户端

- `packages/kilo-vscode/`
  - VS Code 扩展
  - 启动或连接 `kilo serve`
  - 通过 HTTP + SSE 使用 CLI backend

### 其他客户端或外围包

- `packages/app/`: Web UI
- `packages/desktop/`: Tauri 桌面端
- `packages/sdk/js/`: 服务端 API 的自动生成 SDK
- `packages/kilo-gateway/`: Kilo 网关与认证路由
- `packages/kilo-telemetry/`: 遥测
- `packages/kilo-ui/`: UI 组件库
- `packages/kilo-i18n/`: 多语言

## 为什么企业二开要先看 CLI

- 真正的业务能力、工具执行、模型调用、会话状态都在 CLI
- 客户端通常只是把 CLI 能力转成产品体验
- 如果只看扩展 UI，会知道“按钮在哪里”，但不知道“能力从哪来”
