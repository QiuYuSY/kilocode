# CLI 核心模块地图

## 主链路

当前我们已确认的主链路是：

`packages/opencode/src/index.ts`
-> CLI 命令注册与全局初始化
-> `packages/opencode/src/cli/cmd/serve.ts`
-> `packages/opencode/src/server/server.ts`
-> `packages/opencode/src/server/routes/*`
-> `session / tool / provider / project / worktree` 等核心模块

## 模块职责

### `src/index.ts`

- CLI 进程总入口
- 注册命令
- 初始化日志、遥测、全局环境
- 处理一次性数据库迁移
- 在退出时做 telemetry 和 instance 清理

### `src/cli/cmd/serve.ts`

- `kilo serve` 的命令实现
- 解析网络参数
- 启动 HTTP server
- 开发态下同步 workspace
- 处理进程信号与优雅退出

### `src/server/server.ts`

- Hono server 总装配
- 认证、中间件、CORS、错误处理
- 挂载各类 route
- 提供 SSE 事件流
- 通过 `Instance.provide` 和 workspace 上下文绑定当前请求作用域

### `src/cli/network.ts`

- 解析 `serve` 命令的网络参数
- 将命令行参数与全局配置合并
- 是服务部署形态的重要入口

### `src/project/instance.ts`

- 管理按目录维度缓存的实例上下文
- 暴露 `directory`、`worktree`、`project`
- 提供 `Instance.state()` 作为目录级懒单例机制
- 负责实例重载与资源释放

### `src/project/bootstrap.ts`

- 首次 boot 某个目录实例时初始化目录级能力
- 把 plugin、LSP、watcher、VCS、snapshot 等资源挂到实例生命周期里

### `src/control-plane/workspace-*`

- 管理 workspace 级别上下文和请求路由
- 为远端工作区或控制平面能力提供基础

### `src/session/`

- 会话模型
- 消息处理
- prompt 拼装
- 状态、摘要、重试、回滚等逻辑

### `src/tool/`

- 工具定义与注册
- 内置工具和插件工具统一纳管
- 根据模型能力决定使用 `apply_patch` 还是 `edit/write`

### `src/provider/`

- 模型与 provider 抽象
- 使用 AI SDK 适配不同模型提供商
- 模型缓存与鉴权信息联动

### `src/worktree/`

- 工作区隔离基础能力
- 对多会话、并行任务、企业流程隔离非常重要

## 企业二开最可能动到的点

- `server/routes/`: 新增后端 API 或扩展能力
- `provider/`: 接企业内部模型、网关或鉴权逻辑
- `tool/`: 增加工具、限制工具、增加审计
- `session/`: 调整上下文、策略、压缩、记忆
- `worktree/`: 增强隔离、多分支、多环境策略
- `kilocode/`: Kilo 特有扩展能力
