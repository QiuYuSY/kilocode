# 2026-04-07 策略与隔离笔记

## 本轮主题

把企业级二开里另外两个高价值问题先勾出轮廓：

- 策略怎样形成
- 隔离怎样落地

## 新确认的关键点

### 1. config 已经具备企业托管能力

配置并不是只读用户本地文件，而是会合并：

- legacy Kilocode 迁移
- organization modes
- remote config
- global / project / config directories
- managed config

其中 managed config 是最高优先级，这很像企业托管策略入口。

### 2. organization modes 会真正影响最终 agent

它不是 UI 展示层概念，而是会进入 config，再进入 agent 形成链。

### 3. agent 才是组织策略的真正执行载体

以后企业如果要做：

- 部门模式
- 审计模式
- 只读模式
- 指定模型模式

主要应该通过 config 和 agent 合成，而不是直接在 session 里写特殊分支。

### 4. worktree 和 workspace 是两层隔离

- worktree 偏本地 git 隔离
- workspace 偏控制平面和远端抽象

企业后续如果要做本地并行开发和云工作区，大概率两层都要用。

## 对最终目标的帮助

这轮之后，企业接管的地图已经更完整了：

- 模型与网关：provider
- 权限与治理：tool + PermissionNext
- 组织策略：config + agent + org modes
- 隔离与并行：project + instance + workspace + worktree
