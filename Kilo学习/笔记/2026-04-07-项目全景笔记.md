# 2026-04-07 项目全景笔记

## 本轮目标

建立第一层稳定认知：这个仓库的核心在哪里，企业级二次开发应该先看什么。

## 已确认结论

### 1. 这是一个核心平台加多客户端的架构

`packages/opencode/` 不是普通子包，而是整个平台的内核。它负责：

- agent runtime
- server
- session
- tool
- provider
- project / worktree

### 2. VS Code 扩展不是内核，而是 CLI backend 的消费者

`packages/kilo-vscode/` 会启动 `kilo serve --port 0`，然后通过 HTTP + SSE 与其通信。这个认知很关键，因为它决定了企业级改造时优先修改内核，而不是先改 extension。

### 3. `serve` 是非常重要的理解入口

从 `serve` 进入，可以看到：

- server 如何启动
- 请求怎样落入 Hono routes
- 每个请求怎样绑定到 workspace / directory / instance
- SSE 事件流怎样向客户端推送

### 4. 工具系统是产品能力的核心组成

`src/tool/registry.ts` 展示了三类重要能力：

- 内置工具注册
- 插件工具纳入统一体系
- 根据模型能力切换 `apply_patch` 与 `edit/write`

这说明企业二开里常见的工具管控、审计、白名单、能力扩展，都有明确落点。
