## 当前认知

- 仓库是 Bun + Turborepo monorepo
- `packages/opencode/` 是 AI agent runtime、HTTP server、session、tool、provider 的核心实现
- `packages/kilo-vscode/` 是重要客户端，但它本质上是 CLI backend 的消费者
- 企业二开最重要的是理解平台内核和边界，而不是先改 UI

## 当前判断

- 现在已经不需要继续做大范围探索了
- 已经具备进入“系统学习 + 定向补充探索”阶段的条件
- 后续每一轮应该以学习一个主题为主，只在被该主题卡住时做局部探索
- 当前最适合正式开始学习的顺序是：启动与作用域 -> 会话执行链路 -> 模型接入与网关 -> 工具与权限治理 -> 策略形成与隔离
