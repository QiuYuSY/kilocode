### 问题 4：如果用户对 TypeScript 语法和大型项目设计都不熟，应该怎么教？

答案：

- 从真正入口文件开始
- 每次只读一小段真实代码
- 先讲语法，再讲运行时，再讲设计意图
- 在课程里直接放代码片段，并加中文注释和拆解

原因：

- 直接讲抽象架构容易失去抓手
- 只给引用不够，必须把代码摊开讲
- 对大型项目的理解最好建立在一条真实调用链上

### 问题 5：`yargs` 链式调用里的每个方法都是干嘛的？

答案：

- 要逐个解释每个方法的职责，而不是一句“这是链式调用”带过
- 需要区分：
  - 哪些是在定义 CLI 外壳
  - 哪些是在定义全局选项
  - 哪些是在定义执行前钩子

原因：

- 对初学者来说，链式调用很容易看成“魔法”
- 只有把每个方法拆开，才能看懂入口文件到底在配置什么

### 问题 6：为什么 `import { Auth } from "./auth"` 能工作，明明没有 `auth.ts`？

答案：

- 因为这里导入的是模块目录
- `./auth` 会解析到 `./auth/index.ts`
- `./global` 同理会解析到 `./global/index.ts`

原因：

- 大型 TS 项目常常用“目录 + index.ts”来表示一个模块
- 这样模块内部可以继续拆文件，但对外导入路径保持稳定简洁
