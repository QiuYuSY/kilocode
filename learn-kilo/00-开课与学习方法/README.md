# 00 课：开课与学习方法

## 适合谁

- 第一次系统读大型工程的人
- 之前容易在仓库里迷路的人

## 前置要求

- 会基础 Git
- 能看懂 TypeScript 基本语法

## 预计时长

- 总时长：1 到 1.5 小时
- 推荐分 2 次完成

## 你现在的起点

- 你大概率还没有这套仓库的清晰地图
- 你最容易犯的错是直接扑进源码细节

## 这节课新增什么

- 给你一套固定学习动作
- 让你知道每一课该如何判断自己学会了

## 完成线

- 最低完成线：知道整套课程如何使用
- 标准完成线：建立自己的学习模板
- 深入完成线：能解释为什么工程学习必须先建地图

## 难度梯度

1. 观察：接受仓库很大
2. 追链：理解课程怎么安排
3. 解释：说清为什么先学方法
4. 思考：反思自己过去的学习误区

## 源码阅读顺序卡

1. 读顶层 [README.md](/d:/Code/kilocode/learn-kilo/README.md)
2. 读 [课程总表.md](/d:/Code/kilocode/learn-kilo/课程总表.md)
3. 读 [学习路径.md](/d:/Code/kilocode/learn-kilo/学习路径.md)

## 必须关注的结构

- 课程分层
- 每课的 4 个文件职责
- 每课的完成线

## 这课暂时不用深挖

- 任何业务源码
- 任何测试文件

## 在真实开发中有什么用

- 帮你减少在大型仓库里迷路的时间
- 帮你形成稳定的源码阅读方法
- 帮你把“我看过了”变成“我能解释了”

## 常见误区

- 先乱看文件，后面再整理
- 只读不输出
- 觉得眼熟就等于理解

## 本课结论

你接下来不是在“随便翻仓库”，而是在按一套工程课程系统学习。
