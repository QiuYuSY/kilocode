# 00 实验与练习

## 第一阶段: 不看源码完成主任务

## 任务 1: 写下你的旧预期

请先写 5 句话:

- 你原来以为这套课会教你什么
- 你原来以为 KiloCode 是什么项目
- 你最担心自己看不懂的部分是什么
- 你最想二开的能力是什么
- 你现在最想先搞懂哪一条主链

目的:

把旧预期显性化，后面才知道自己真正修正了什么。

## 任务 2: 先画“空白地图”

不要查资料，先凭你现在的理解画一张图:

- 核心运行时在哪里
- 扩展端在哪里
- SDK 在哪里
- Kilo 差异层在哪里

目的:

后面每学一课，都能和这张初始图比较。

## 任务 3: 写下你的毕业目标

从下面 3 类里选一个:

- 我想做一个 CLI / Server / Tool 侧的小改动
- 我想做一个 VS Code 扩展侧的小改动
- 我想做一个 Kilo 平台差异相关的小改动

然后补一句:

“为了做到这件事，我最需要搞懂的是 ________。”

## 任务 4: 建立证据链笔记页

单独建一页学习笔记，后面每节课统一记录 5 件事:

- 本课主链
- 本课关键文件
- 本课核心抽象
- 本课最重要的设计判断
- 本课未来二开落点

## 第二阶段: 回到课程入口和主教材做校准

## 验证 1: 浏览主教材和任务列表

请再去看:

- `KiloCode架构总册.md`
- `KiloCode二开手册.md`
- `实施任务列表.md`

然后补一句:

“我原来的学习预期，和这套课真实目标之间最大的差距是 ________。”

## 验证 2: 修正你的空白地图

根据主教材，把你第一版空白地图补成第二版。

## 完成标准

- 你已经写出旧预期和毕业目标
- 你已经画出第一版空白地图
- 你已经用主教材把第一版地图修正成第二版
- 你已经为后面的课程准备好统一笔记格式
