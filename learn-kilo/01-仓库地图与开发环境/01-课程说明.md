# 01 仓库地图与开发环境

## 本课要解决的问题

这一课不是教你认目录，而是让你建立一个足够稳定的系统地图。

学完这一课后，你应该不再把 KiloCode 理解成:

- 一个 VS Code 插件仓库
- 一个单体应用
- 一个“目录很多但关系不清楚”的项目

你应该开始把它理解成:

核心运行时 + 多客户端接入 + 契约共享层 + Kilo 平台差异层。

## 本课能力目标

- 能说清主要包的职责边界
- 能说清为什么 `packages/opencode` 是第一优先级
- 能说清开发时最重要的命令和工作约束
- 能建立第一次“需求 -> 包级落点”的判断

- 你能画出仓库四层图

- 你知道常用命令、测试入口和 merge 友好约束
