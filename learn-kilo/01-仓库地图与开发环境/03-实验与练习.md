# 01 实验与练习

## 第一阶段: 不看源码完成主任务

## 任务 1: 画四层结构图

请只根据课程讲义和你的理解，先画出四层结构图:

- 核心运行时层
- 客户端接入层
- 契约与共享层
- Kilo 平台差异层

要求:

- 每层至少填 2 个真实包或目录
- 每层写一句“它主要解决什么问题”

## 任务 2: 做一次“需求归层”

把下面 6 个需求分别归到你认为最先应该改的层:

1. 给 Agent 新增一个工具
2. 增加一个后端 route
3. 让扩展端多一个按钮
4. 修改 Kilo 的鉴权头部
5. 调整共享 UI 组件
6. 新增会话事件展示

要求:

- 每个需求都写“为什么先改这里，不先改别处”

## 任务 3: 口头讲清系统主体

请脱稿回答:

“为什么 KiloCode 的系统主体是核心运行时，而不是 VS Code 扩展？”

要求:

- 控制在 2 分钟以内
- 不允许引用源码文件名
- 只能用工程结构和职责边界来解释
