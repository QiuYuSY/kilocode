# 01 课：课程准备与仓库地图

## 这节课到底在解决什么

这节课不是“开场白”，而是整套课程最重要的地基。

如果第一课只让你知道几个目录名字，那它就是失败的。  
这一课真正要解决的是：

- 让你第一次建立对当前仓库的全局地图
- 让你知道哪些包是核心，哪些包是客户端，哪些包是共享层
- 让你知道为什么后面课程必须先学核心运行时，再学扩展和 UI
- 让你知道以后读源码时应该如何判断“我现在看到的东西在系统里属于哪一层”

学完这一课后，你不应该再把这个仓库看成“很多目录”，而应该开始把它看成“一个核心引擎 + 多个客户端 + 一组共享桥梁 + 一层 Kilo 增量能力”的系统。

## 本课学习目标

完成本课后，你至少要能做到：

1. 说清楚这个仓库有哪些主要产品和主要包。
2. 说清楚为什么 [packages/opencode](/d:/Code/kilocode/packages/opencode) 是整个系统最值得先看的位置。
3. 说清楚为什么 [packages/kilo-vscode](/d:/Code/kilocode/packages/kilo-vscode) 不是“系统主体”，而是一个重要客户端。
4. 说清楚为什么 [packages/sdk/js](/d:/Code/kilocode/packages/sdk/js) 不是边角料，而是多客户端共享接口的桥梁。
5. 画出你自己的“仓库分层图”和“产品关系图”。

## 建议学习时长

- 预估总时长：2 到 3 小时
- 推荐拆成 3 段：
  1. 30 到 40 分钟读讲义
  2. 40 到 60 分钟做实验
  3. 30 到 40 分钟做复盘和输出

## 先修要求

你不需要会这个仓库里的业务逻辑，但建议具备：

- 基础 Git 使用能力
- 基础 TypeScript 阅读能力
- 能看懂 `package.json`
- 对 monorepo、CLI、HTTP 这些词至少不陌生

## 这一课你先不要追什么

这一课明确不追下面这些内容：

- 不追任何具体业务实现细节
- 不追 Session、Agent、Tool 的内部逻辑
- 不追 UI 组件实现
- 不追测试

你的任务是“搭地图”，不是“解剖器官”。

## 学完后你应该留下什么

本课结束时，你至少要留下 3 个东西：

1. 一张仓库分层图
2. 一份“核心包优先级清单”
3. 一段 2 到 3 分钟的仓库介绍稿

## 通过标准

如果你能不看笔记讲清下面 4 个问题，就算这一课过关：

1. 这个仓库有哪些主要产品？
2. 为什么 [packages/opencode](/d:/Code/kilocode/packages/opencode) 是第一优先级？
3. 为什么 [packages/kilo-vscode](/d:/Code/kilocode/packages/kilo-vscode) 不能单独理解？
4. 为什么后面的课程顺序应该是“先核心运行时，再客户端，再 Kilo 差异”？
