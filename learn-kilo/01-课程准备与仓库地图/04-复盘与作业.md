# 复盘与作业

## 自测问题

请先独立回答下面 8 个问题，不要急着看讲义。

1. 这个仓库更像单产品仓库，还是多产品 monorepo？
2. 为什么 [packages/opencode](/d:/Code/kilocode/packages/opencode) 是学习起点？
3. 为什么 [packages/kilo-vscode](/d:/Code/kilocode/packages/kilo-vscode) 不能单独理解？
4. [packages/sdk/js](/d:/Code/kilocode/packages/sdk/js) 为什么不是边缘包？
5. 当前仓库里哪些包更像客户端？
6. 当前仓库里哪些包更像共享桥梁？
7. Kilo 的增量能力主要落在哪些地方？
8. 为什么后面课程应该先讲核心，再讲客户端？

## 本课作业

这一课必须交 3 项成果：

### 作业 1：仓库分层图

要求：

- 至少包含 8 个关键包
- 明确分层
- 明确关系

### 作业 2：核心包优先级清单

要求：

- 至少写出 8 个包
- 每个包都写一句理由

### 作业 3：仓库介绍稿

要求：

- 时长 2 到 3 分钟
- 能讲给新人听
- 不要照抄讲义句子

## 优秀答案示例结构

### 仓库分层图

- 第一层：核心引擎
- 第二层：客户端
- 第三层：共享桥梁
- 第四层：Kilo 增量

### 核心包优先级清单

- 包名
- 处于哪一层
- 为什么当前优先级高或低

### 仓库介绍稿

- 先说这是什么类型的仓库
- 再说核心是谁
- 再说客户端有哪些
- 最后说为什么学习顺序是先核心后客户端

## 验收标准

如果你满足下面 4 条，这节课就算通过：

1. 你能画出一张有层次的仓库图，而不是只列目录名。
2. 你能清楚说出为什么 [packages/opencode](/d:/Code/kilocode/packages/opencode) 是第一优先级。
3. 你能解释为什么 [packages/kilo-vscode](/d:/Code/kilocode/packages/kilo-vscode) 是客户端而不是系统主体。
4. 你能说明后面课程为什么必须先讲核心运行时。

## 不通过的典型表现

如果你出现下面这些情况，说明这一课还没学扎实：

- 你还把仓库理解成“主要是个 VS Code 插件”
- 你觉得每个包差不多重要
- 你解释不清 SDK 为什么存在
- 你画不出层级关系，只能列目录

## 本课结束后的理想状态

到了这里，你还不需要懂业务代码。

但你应该已经有一种很重要的能力：

当你看到一个新目录时，你会先问自己：

- 它属于哪一层？
- 它更接近核心还是更接近客户端？
- 它是能力实现还是共享桥梁？

只要这个能力立住了，后面课程就会顺很多。
