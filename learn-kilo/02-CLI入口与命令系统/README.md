# 02 课：CLI 入口与命令系统

## 适合谁

- 想从最真实入口进入核心代码的人

## 前置要求

- 完成 01 课

## 预计时长

- 总时长：2 到 3 小时
- 推荐分 2 到 3 次完成

## 你已经学会了什么

- 你知道整个仓库的产品关系
- 你知道 `packages/opencode` 是核心引擎

## 这节课新增什么

- 真实入口
- 命令注册链路
- 入口文件的全局编排职责

## 完成线

- 最低完成线：知道 CLI 从哪里启动
- 标准完成线：能说清命令注册链路
- 深入完成线：能解释入口文件的全局编排职责

## 难度梯度

1. 观察：认入口
2. 追链：认命令注册
3. 解释：理解入口初始化职责
4. 思考：理解为什么命令层要这样组织

## 源码阅读顺序卡

1. `packages/opencode/src/index.ts`
2. 只看 `.command(...)` 注册列表
3. `packages/opencode/src/cli/cmd/cmd.ts`
4. `packages/opencode/src/cli/cmd/serve.ts`
5. 再回 `index.ts` 看 middleware 与 finally

## 必须关注的函数/结构

- `yargs(...)`
- `.command(...)`
- `ServeCommand`
- 入口里的 middleware
- 入口里的 finally 清理

## 这课暂时不用深挖

- 每个具体命令的全部实现细节
- 非关键命令如 `db`、`export` 的全部逻辑

## 在真实开发中有什么用

- 帮你定位 CLI 启动问题
- 帮你快速找到新命令应插入的位置
- 帮你理解全局初始化逻辑为什么会影响所有客户端

## 常见误区

- 把入口文件当成“只是注册命令”
- 以为每个命令都同等重要
- 忽略 finally 清理对长生命周期进程的重要性

## 本课结论

CLI 入口文件不是简单启动器，而是全局初始化和命令编排中心。

## 下一课为什么学它

命令系统看懂后，下一步自然是进入真正承接这些命令的后端主干：Server。
