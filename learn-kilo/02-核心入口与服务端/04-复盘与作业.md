## 优秀答案示例结构

### CLI -> Server 主链路图

- `index.ts`
- 命令层
- `ServeCommand`
- `Server.listen`
- `/event`

### 入口文件职责说明

- 命令注册
- 全局初始化
- 数据迁移
- 错误处理
- finally 清理

### “为什么 `serve` 是系统中枢”

- 先讲它让核心引擎服务化
- 再讲它把多个客户端接到同一后端能力上
- 最后讲它为什么必须认真处理生命周期

## 验收标准

如果你满足下面 4 条，这节课就算通过：

1. 你能清楚讲出入口文件不只是“程序开始执行的地方”，而是在统一初始化系统。
2. 你能清楚讲出 `serve` 为什么让核心引擎获得了“服务端形态”。
3. 你能清楚讲出 `server.ts` 同时在做安全、上下文、能力、实时同步四层事情。
4. 你能解释为什么 SSE 对这个项目不是可选装饰，而是核心机制。

## 不通过的典型表现

如果学习者出现下面这些情况，说明这一课还没学扎实：

- 只知道 `index.ts` 里注册了很多命令
- 说不出 `serve` 和普通命令的差异
- 把 `server.ts` 只理解成接口文件
- 说不清 `/event` 存在的必要性

## 本课结束后的理想状态

到了这里，学习者还不需要懂 Session、Agent、Tool 的内部逻辑。

但必须已经建立下面这个关键认识：

这个仓库的核心不是“很多功能命令”，而是“一个可以既作为 CLI 运行、也可以作为服务端运行的核心引擎”。

这个认识一旦建立，后面再进入 Project、Config、Session 时，就不会把它们看成零散模块。
