# 02 核心运行时总链路

## 本课要解决的问题

这一课开始真正进入 KiloCode 的主链。

我们只抓一条最重要的线:

`index.ts -> serve.ts -> server.ts -> Instance -> Session`

你不需要在这一课读完整个后端，但必须搞清楚:

- 核心运行时如何变成一个 headless server
- 为什么客户端最终都要连接 `kilo serve`
- 服务端如何建立上下文边界和实时事件边界

## 本课能力目标

- 能解释 CLI 入口做了哪些全局初始化
- 能解释 `serve` 为什么不是普通命令
- 能解释 `server.ts` 为什么是系统中枢
- 能解释 `directory` 为什么会一路走到 `Instance.provide(...)`

## 必看文件

- `packages/opencode/src/index.ts`
- `packages/opencode/src/cli/cmd/serve.ts`
- `packages/opencode/src/server/server.ts`
- `packages/opencode/src/project/project.ts`
- `packages/opencode/src/project/instance.ts`

## 本课通过标准

- 你能画出服务端启动链
- 你能解释请求上下文是怎么建立的
- 你能说清为什么没有 SSE 很多体验就无法成立
