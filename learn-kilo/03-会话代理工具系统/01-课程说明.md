# 03 会话代理工具系统

## 本课要解决的问题

这一课是整个课程最核心的一节。

因为真正让 KiloCode 变成“AI 编程执行系统”的，不是网页，不是扩展，而是下面这些运行时抽象:

- Session
- Message / Part
- SessionProcessor
- Agent
- Permission
- ToolRegistry

如果这些东西只停留在“我见过这些文件”，你依然做不了二开。

## 本课能力目标

- 能解释为什么 Session 不是聊天记录
- 能解释为什么 Message 要拆成 Part
- 能解释 `SessionProcessor` 为什么必须存在
- 能解释 Agent 行为边界是怎么由 prompt、permission、tool 共同定义的
- 能解释 ToolRegistry 为什么是动态编排层

## 必看文件

- `packages/opencode/src/session/index.ts`
- `packages/opencode/src/session/message-v2.ts`
- `packages/opencode/src/session/processor.ts`
- `packages/opencode/src/agent/agent.ts`
- `packages/opencode/src/permission/next.ts`
- `packages/opencode/src/tool/registry.ts`

## 本课通过标准

- 你能把一次工具调用在会话里的生命周期讲明白
- 你能讲清 Agent / Permission / Tool 的协作关系
- 你已经具备做小型工具或行为改动的理解基础
