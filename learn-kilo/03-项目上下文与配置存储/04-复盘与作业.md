# 复盘与作业

## 自测问题

请先独立回答下面 8 个问题：

1. 为什么 [packages/opencode/src/project/project.ts](/d:/Code/kilocode/packages/opencode/src/project/project.ts) 不只是路径工具？
2. `fromDirectory(...)` 真正解决的是什么问题？
3. 为什么 `Instance` 不是普通单例包装器？
4. `Instance.state(...)` 相比模块级缓存多提供了什么能力？
5. 为什么配置来源多，不一定说明设计差？
6. 为什么 [packages/opencode/src/config/config.ts](/d:/Code/kilocode/packages/opencode/src/config/config.ts) 会和 `Instance` 绑定？
7. 为什么数据库和文件存储会一起存在？
8. 如果把当前系统改成“全局单例 + 一个 config.json + 全部进数据库”，最先会坏掉什么？

- 说明文件存储适合什么

## 验收标准

如果学习者满足下面 4 条，这节课就算通过：

1. 能清楚解释 `Project` 解决的是“身份识别”问题。
2. 能清楚解释 `Instance` 解决的是“按目录隔离状态”问题。
3. 能清楚解释 `Config` 为什么会有多来源合并。
4. 能清楚解释为什么存储会分层，而不是单一方案统治全局。

## 不通过的典型表现

如果学习者出现下面这些情况，说明这一课还没学扎实：

- 把 project id 直接理解成当前路径
- 把 `Instance` 当成普通全局缓存
- 觉得配置复杂就一定是设计不好
- 认为数据库足够了，文件存储一定多余

## 本课结束后的理想状态

到了这里，学习者还不需要懂 Session 执行循环。

但必须已经建立一个关键认识：

这个系统不是在“全局状态里跑一些命令”，而是在“按项目上下文组织一整套运行时”。

这个认识一旦建立，下一课进入 Session 时，学习者就更容易理解为什么会话、消息、执行状态都必须和项目上下文挂在一起。
