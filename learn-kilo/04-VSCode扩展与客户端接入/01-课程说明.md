# 04 VSCode扩展与客户端接入

## 本课要解决的问题

这一课专门解决一个高频误解:

很多人一看到 KiloCode 的日常使用界面在 VS Code 里，就会自然把扩展当成系统主体。

这是不对的。

这一课要把扩展端拆成 3 层:

- extension host
- CLI backend 连接层
- webview UI

## 本课能力目标

- 能解释 `extension.ts` 是装配入口，不是业务核心
- 能解释 `ServerManager` 如何拉起本地 CLI backend
- 能解释 `KiloConnectionService` 为什么必须共享连接
- 能解释 Agent Manager 为什么是“多会话编排”而不是“另一套后端”

## 必看文件

- `packages/kilo-vscode/src/extension.ts`
- `packages/kilo-vscode/src/services/cli-backend/server-manager.ts`
- `packages/kilo-vscode/src/services/cli-backend/connection-service.ts`
- `packages/kilo-vscode/src/KiloProvider.ts`
- `packages/kilo-vscode/webview-ui/`

## 本课通过标准

- 你能画出扩展接入链
- 你能解释共享连接的必要性
- 你知道扩展侧的改动边界该怎么切
