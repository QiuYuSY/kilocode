# 05 Kilo差异层与平台能力

## 本课要解决的问题

这一课专门回答:

KiloCode 到底比 upstream OpenCode 多了什么，而且这些差异为什么要这样组织。

如果你对这部分没有判断，做二开时就会出现两个风险:

- 明明是 Kilo 专属能力，却改进共享上游文件里
- 明明只是客户端体验，却误改到平台差异层

## 本课能力目标

- 能识别 Kilo 平台差异主要集中在哪些包和目录
- 能解释 `kilocode_change` 的工程意义
- 能解释 config 里 legacy Kilocode 迁移为什么复杂
- 能解释 Kilo Gateway 和 Telemetry 在整体系统里的位置
- 能判断一个需求是否应优先落到 Kilo 专属路径

## 必看文件

- `packages/opencode/src/index.ts`
- `packages/opencode/src/server/server.ts`
- `packages/opencode/src/config/config.ts`
- `packages/opencode/src/kilocode/`
- `packages/kilo-gateway/src/index.ts`
- `packages/kilo-telemetry/src/index.ts`

## 本课通过标准

- 你能清楚指出 Kilo 差异层的几个核心落点
- 你能解释为什么这些能力不应该散着改
- 你知道什么时候必须考虑 `kilocode_change`
