# 06 课：VS Code 扩展全链路

## 这节课到底在解决什么

到了第五课，学习者已经知道：

- 核心引擎如何启动
- 服务端如何成型
- 项目上下文和配置如何建立
- 会话系统如何承载执行状态
- Agent、permission、Tool 如何共同决定“系统能做什么”

但如果此时去看 [packages/kilo-vscode](/d:/Code/kilocode/packages/kilo-vscode)，学习者仍然很容易产生一个巨大误解：

- “这个扩展看起来很完整，那它是不是整个系统主体？”

第六课就是专门纠正这个误解的。

这一课要让学习者真正理解：

1. VS Code 扩展不是后端，而是重要客户端。
2. `extension host`、`webview`、`CLI backend` 是三层不同职责。
3. `ServerManager` 和 `KiloConnectionService` 是扩展端最关键的两个中枢服务。
4. Agent Manager 不是独立产品，而是建立在共享后端上的多会话编排能力。

如果这一课讲不好，学习者后面看扩展代码时会一直误判：

- 把 extension host 当成后端逻辑主体
- 把 webview 当成普通前端应用
- 把 Agent Manager 看成“另一套系统”

这三种误解都会直接妨碍后续理解。

## 本课学习目标

完成本课后，学习者至少要能：

1. 说清楚 [packages/kilo-vscode/src/extension.ts](/d:/Code/kilocode/packages/kilo-vscode/src/extension.ts) 在系统里扮演什么角色。
2. 说清楚 [packages/kilo-vscode/src/services/cli-backend/server-manager.ts](/d:/Code/kilocode/packages/kilo-vscode/src/services/cli-backend/server-manager.ts) 负责什么。
3. 说清楚 [packages/kilo-vscode/src/services/cli-backend/connection-service.ts](/d:/Code/kilocode/packages/kilo-vscode/src/services/cli-backend/connection-service.ts) 为什么比普通 HTTP client 更重要。
4. 说清楚 [packages/kilo-vscode/webview-ui/src/App.tsx](/d:/Code/kilocode/packages/kilo-vscode/webview-ui/src/App.tsx) 处于哪一层、负责什么。
5. 说清楚 Agent Manager 为什么能多会话但只用一个后端。

## 这一课明确不追什么

为了让学习者不被扩展端体量压垮，本课刻意不追：

- 所有 VS Code command 的完整实现
- 所有 webview 组件细节
- 所有 Agent Manager UI 交互实现
- 所有自动补全、浏览器自动化、commit message 细节

本课只回答：

- “扩展怎样接入核心引擎”
- “三层职责怎样分开”
- “Agent Manager 如何建立在共享后端之上”
