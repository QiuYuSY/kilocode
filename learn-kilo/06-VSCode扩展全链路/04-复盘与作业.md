# 04-复盘与作业

## 自测问题

请先独立回答下面 8 个问题：

1. 为什么 VS Code 扩展不是系统主体？
2. 为什么扩展端必须自己拉起一个 CLI backend？
3. `ServerManager` 和“直接 exec 一个命令”有什么工程上的区别？
4. `KiloConnectionService` 为什么不能被普通 HTTP client 替代？
5. 为什么共享连接对多面板 / 多会话场景如此重要？
6. 为什么 webview 不能被理解成普通 web app？
7. Agent Manager 为什么不是另一套独立系统？
8. 如果每个 panel 都自己启动一个后端，最先会坏掉什么？

## 验收标准

如果学习者满足下面 4 条，这节课就算通过：

1. 能清楚解释扩展是客户端而不是系统主体。
2. 能清楚解释 `ServerManager` 负责后端进程生命周期。
3. 能清楚解释 `KiloConnectionService` 负责连接与事件编排。
4. 能清楚解释 Agent Manager 建立在共享后端之上。

## 不通过的典型表现

如果学习者出现下面这些情况，说明这一课还没学扎实：

- 把 extension host 当成核心执行层
- 把 webview 当成完整前端应用
- 把 Agent Manager 看成另一套后端
- 说不清为什么连接必须共享

## 本课结束后的理想状态

到了这里，学习者还不需要吃透 SDK 和共享前端层细节。

但必须已经建立一个关键认识：

VS Code 扩展的价值，不在于它自己“做了全部逻辑”，而在于它把核心引擎稳定地接进了一个编辑器环境，并把连接、事件、UI、会话编排组织了起来。

这个认识一旦建立，下一课进入 SDK、共享前端与 Kilo 差异时，学习者就更容易理解哪些是“客户端共性”，哪些是“平台特性”。
