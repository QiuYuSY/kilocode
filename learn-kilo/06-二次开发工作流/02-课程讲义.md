# 06 课程讲义

## 一、二开时先做的不是写代码，而是判层

面对一个需求，先问 4 个问题:

1. 这是核心运行时能力吗
2. 这是客户端接入能力吗
3. 这是契约层问题吗
4. 这是 Kilo 专属平台差异吗

如果这一层判断错了，后面的改动很容易越改越乱。

## 二、5 类常见需求的标准落点

### 1. 新增工具

通常先看:

- `packages/opencode/src/tool/`
- `packages/opencode/src/tool/registry.ts`
- `packages/opencode/src/agent/agent.ts`
- `packages/opencode/src/permission/next.ts`

### 2. 新增 route

通常先看:

- `packages/opencode/src/server/routes/`
- `packages/opencode/src/server/server.ts`

然后判断:

- 客户端是否会调用
- 是否要跑 SDK regen

### 3. 调整 Agent 行为

通常先看:

- `packages/opencode/src/agent/agent.ts`
- 相关 prompt
- `permission/next.ts`

### 4. 扩展端接入新能力

通常先看:

- 后端 route / event 有没有
- `packages/kilo-vscode/src/services/cli-backend/`
- 对应 provider / webview UI

### 5. Kilo 专属能力

优先看:

- `packages/opencode/src/kilocode/`
- `packages/kilo-gateway/`
- `packages/kilo-telemetry/`

## 三、最小改动路径怎么设计

一个健康的小改动路径一般是:

1. 先确认真正的能力边界
2. 只改必须改的主链节点
3. 如果接口变了，再补 SDK / client
4. 最后补验证

要避免:

- 还没定位就先写 UI
- 共享文件能不改却硬改
- 没有验证链就直接扩散改动

## 四、fork 仓库里最重要的工作流意识

因为 KiloCode 是 fork，所以二开不能只追求“能跑”。

还要考虑:

- 和 upstream merge 的冲突成本
- `kilocode_change` 的标记边界
- Kilo 专属代码的集中放置

## 五、一个需求拆解示例

例如: “给 Agent 增加一个新的只读分析工具”

你至少要想清:

- 工具本体落在哪个文件
- ToolRegistry 如何注册
- 哪些 Agent 能用
- Permission 怎么限制
- Session / UI 是否需要展示新 Part
- 是否要补测试

这才叫完整二开思维，而不是“先把函数写出来”。

## 本课最重要的设计判断

二开工作流的起点不是编码速度，而是落点判断和边界控制。

## 脱离源码版主讲

如果你不看任何代码，也应该先学会这一课最关键的动作链:

拿到一个需求后，不是先写代码，而是先判断:

1. 它属于哪一层
2. 它会影响哪条主链
3. 它最小要改哪些节点
4. 它的风险是什么
5. 怎么做最小验证

只要这 5 步能稳定走下来，你就已经具备了真实的二开工作流意识。

## 关键抽象解剖卡

### 抽象卡 1: 判层

它是什么:

先判断需求属于哪一层的动作。

为什么必须存在:

因为落点判断错了，后面所有实现都会偏。

没有它会怎样:

最常见的结果就是先改 UI、后补主链，最终边界混乱。

### 抽象卡 2: 最小改动路径

它是什么:

一个需求从目标到改动文件再到验证方式的最短路径。

为什么必须存在:

因为 fork 项目最怕无边界扩散改动。

没有它会怎样:

你会越改越多，且说不清为什么。

### 抽象卡 3: 风险清点

它是什么:

在动手前先识别 SDK、扩展、shared file、Kilo 差异层等风险点。

为什么必须存在:

因为 KiloCode 不是单包项目，波及面比你想象的大。

没有它会怎样:

改动表面完成，交付时才暴露问题。

## 常见二开场景

### 场景 1: “给工具系统加一个新能力”

你应该先想到:

- tool
- registry
- agent
- permission

### 场景 2: “给服务端加一个新接口并让扩展调用”

你应该先想到:

- server route
- SDK 契约
- 扩展连接层 / webview

### 场景 3: “给 Kilo 平台做一个组织级能力”

你应该先想到:

- 这是 Kilo 差异层，不是共享层
