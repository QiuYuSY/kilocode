# 06 实验与练习

## 第一阶段: 不看源码完成主任务

## 任务 1: 需求拆路径

请从下面 5 个题目里任选 3 个，分别写出最小改动路径:

1. 新增一个搜索型工具
2. 新增一个 session 相关 route
3. 给扩展端增加一个“查看最近会话状态”的入口
4. 给 Kilo 平台增加一个自定义 header
5. 给一个 Agent 增加新的只读权限规则

要求:

- 至少写出会改哪些包
- 至少写出会改哪些关键文件
- 至少写出一个主要风险点

## 任务 2: 做一次“是否需要 regen SDK”判断

请对下面情况做判断:

- 只改服务端内部逻辑
- 新增 route
- 修改 route 的请求体
- 只改扩展端 UI

## 任务 3: 做一次“是否属于 Kilo 差异层”判断

请对下面情况做判断:

- 加 organization modes 拉取
- 调整通用 read tool
- 修改 Kilo telemetry 事件
- 给扩展端多一个按钮

## 第二阶段: 回到源码做验证

## 验证 1: 用真实文件落地你的一条改动路径

从你在任务 1 里写的 3 条改动路径中任选 1 条，回到真实仓库，验证:

- 这些包是否真的是主要落点
- 这些关键文件是否确实在主链上

## 验证 2: 检查你的风险判断是否充分

请特别检查你是否考虑了:

- SDK
- Kilo 差异层
- shared file
- 客户端接入

## 输出练习

写一份不超过 1 页的“小改动实施方案”，题目自选，但必须包含:

- 目标
- 改动落点
- 风险
- 验证方式

## 完成标准

- 你已经能把需求拆成改动路径
- 你已经能先靠课程本体写出方案，再用源码校验
- 你已经开始具备真正的二开方案意识
