# 06 复盘与作业

## 复盘问题

1. 为什么二开时第一步是判层而不是写代码
2. 为什么 route 改动要考虑 SDK
3. 为什么 fork 仓库里改动边界比单仓库更重要
4. 你最容易在哪种需求上误判落点
5. 你现在做一个小改动时，最担心的风险是什么

## 本课作业

提交 3 个结果物:

- 3 个需求的最小改动路径
- 一页小改动实施方案
- 一份“是否需要 SDK regen / 是否属于 Kilo 差异层”的判断表

要求:

- 先按课程本体写方案
- 再至少拿一条路径去真实仓库验证落点

## 本课验收标准

- 你已经会从需求倒推改动落点
- 你已经能控制改动范围
- 你已经开始具备真实交付前的方案意识
- 你已经能把课程里的判断落实到真实实现节点
