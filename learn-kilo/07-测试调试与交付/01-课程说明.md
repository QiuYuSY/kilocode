# 07 测试调试与交付

## 本课要解决的问题

会改，不等于能交付。

这一课解决的是:

你做完一个改动之后，如何在 KiloCode 这样的 monorepo + fork 项目里完成基础验证、调试和交付。

## 本课能力目标

- 知道不同包的验证入口
- 知道 extension、CLI、SDK 相关改动分别怎么验
- 知道哪些改动必须补额外生成或检查
- 知道如何减少后续 merge 和 CI 风险

## 本课重点命令

- `bun turbo typecheck`
- `bun test` from `packages/opencode/`
- `bun test test/...`
- `./script/generate.ts`
- `bun run script/extract-source-links.ts`
- `bun run check-kilocode-change`
- `bun run extension`

## 本课通过标准

- 你能为一个小改动列出最小验证清单
- 你知道不同类型改动各自要跑什么
- 你知道交付前该检查哪些 fork 约束
