# 08 课程讲义

## 一、什么样的毕业题目才是好题目

好题目不是“大”，而是“小而完整”。

### 好题目的特征

- 边界清楚
- 主链明确
- 可以用 5 到 10 个关键文件支撑
- 可以给出明确验证方式

### 坏题目的特征

- 想一次讲完整个仓库
- 想一次做很大的功能
- 说不清改动边界
- 没法验证

## 二、推荐的毕业题方向

### 方向 1: 拆一条主链

例如:

- 从 `serve` 到 `server` 的服务化主链
- 从扩展到 CLI backend 的接入主链
- 从工具调用到 Session Part 的执行主链

### 方向 2: 做一个小型功能改动

例如:

- 给一个工具加能力
- 给 Agent 增加一个新的只读行为边界
- 给扩展端增加一个小入口

### 方向 3: 做一个 Kilo 差异层分析或改动

例如:

- 分析 Kilo config 迁移链
- 分析 Gateway / Telemetry 接入链
- 对 Kilo 专属能力做一次小型改造

## 三、毕业成果必须长什么样

最少包含 6 部分:

1. 题目与范围
2. 关键源码证据链
3. 设计解释
4. 改动方案或实现
5. 验证方法与结果
6. 复盘

## 四、评分时最看重什么

最看重的不是字数，而是:

- 证据链是否真实
- 边界是否控制住
- 设计解释是否讲到位
- 验证是否靠谱

## 本课最重要的设计判断

真正能证明你学会了，不是你写了很多，而是你已经能把理解组织成别人也能复用的成果。

## 脱离源码版主讲

毕业项目真正考的不是“你有没有继续看更多代码”，而是:

你能不能把前面学到的系统理解和二开方法，组织成一个完整、可信、可检验的成果。

所以毕业项目最重要的不是大，而是完整:

- 题目清楚
- 边界清楚
- 证据链清楚
- 设计判断清楚
- 验证清楚

## 关键抽象解剖卡

### 抽象卡 1: 题目边界

它是什么:

你这次毕业项目明确要做和不做的范围。

为什么必须存在:

因为题目一旦发散，成果就会失焦。

没有它会怎样:

你会做成“大而空”的总结。

### 抽象卡 2: 源码证据链

它是什么:

支撑你结论和方案的关键文件与关键主链。

为什么必须存在:

因为课程最终要证明的是你真的理解了工程，而不是写主观感受。

没有它会怎样:

成果会失去可信度。

### 抽象卡 3: 可检验成果

它是什么:

别人看完后能判断你是不是真的掌握了 KiloCode 的那份产物。

为什么必须存在:

因为“自我感觉学会了”不算课程终点。

没有它会怎样:

毕业只是形式，不是能力证明。

## 常见二开场景

### 场景 1: 做架构解剖型项目

重点:

- 主链要清楚
- 关键抽象要讲透

### 场景 2: 做小型真实改动型项目

重点:

- 改动边界要小
- 风险和验证要完整

### 场景 3: 做客户端接入型项目

重点:

- 分清 extension host、连接层、webview

## 源码映射附录

毕业项目阶段回到源码，不是为了重新学习，而是为了:

- 建立证据链
- 确认改动落点
- 验证你的结论和方案

所以在这一步，代码页已经变成你的证据库，而不是你的老师。
