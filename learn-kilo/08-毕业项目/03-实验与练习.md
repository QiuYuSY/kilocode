# 03-实验与练习

## 任务 1：先缩题，不要先写

请先从下面两类中选一个方向：

### A 类：源码讲解型

候选示例：

- `kilo serve` 从启动到服务成型
- Session / Message / Part 模型设计
- 一次 tool call 生命周期
- VS Code 扩展连接 CLI backend 链路

### B 类：小型真实改动型

候选示例：

- 一处小文案
- 一处小日志
- 一处空状态
- 一处轻量 bug 修复
- 一处说明型配置增强

要求：

- 先写题目
- 再写为什么选它
- 再写为什么它的范围足够小

## 任务 2：建立证据链清单

无论你选哪种题目，都必须先列出：

1. 入口文件
2. 关键模块
3. 关键函数 / 结构
4. 关键设计点

要求：

- 每一项都必须写真实路径
- 每一项都要写一句“为什么它重要”

## 任务 3：写一个最小大纲

### 如果你做讲解型

请先写最小大纲：

- 背景
- 入口
- 主链路
- 关键模块
- 设计点
- 我的理解

### 如果你做改动型

请先写最小大纲：

- 问题是什么
- 证据是什么
- 涉及哪些文件
- 为什么改这个点
- 怎么验证

## 任务 4：形成成果初稿

最少完成其中一项：

- 一份源码讲解初稿
- 一份改动说明初稿

如果你精力允许，建议同时做两项。

## 任务 5：写复盘

复盘至少回答：

1. 我现在最能讲清的 3 个点
2. 我还不够扎实的 3 个点
3. 如果继续深入，我最该补哪一层

## 口头练习

请尝试脱稿讲 3 分钟：

“为什么我选这个题目，以及我打算怎样证明自己真的理解了它？”

## 任务记录模板

你可以按下面结构写：

- 我的题目
- 我为什么选它
- 我的证据链
- 我的最小大纲
- 我的成果状态
- 我的复盘
