# 09 课：Tool 注册与动态编排

## 适合谁

- 已理解 Agent 模式，想知道具体工具如何进入运行时的人

## 前置要求

- 完成 08 课

## 预计时长

- 总时长：2 到 3 小时
- 推荐分 2 到 3 次完成

## 你已经学会了什么

- 你已经知道模式的边界如何决定 agent 能做什么
- 你知道 permission 会影响行为范围

## 这节课新增什么

- 工具注册
- 动态工具集
- 模型与插件对工具的影响

## 完成线

- 最低完成线：知道 `ToolRegistry` 的作用
- 标准完成线：能追踪一个工具从定义到暴露给模型
- 深入完成线：能解释为什么工具集必须动态编排

## 难度梯度

1. 观察：认工具清单
2. 追链：认一个工具的注册路径
3. 解释：认模型、配置、插件如何影响工具集
4. 思考：比较静态工具集与动态工具集

## 源码阅读顺序卡

1. `packages/opencode/src/tool/registry.ts`
2. 看 `all()`
3. 看 `tools(...)`
4. 选一个工具，如 `read.ts` 或 `bash.ts`
5. 再看 `apply_patch.ts`

## 必须关注的函数/结构

- `ToolRegistry.all`
- `ToolRegistry.tools`
- `Tool.Info`
- `fromPlugin`

## 这课暂时不用深挖

- 每个工具的全部内部实现
- 所有实验工具开关

## 在真实开发中有什么用

- 帮你定位某个工具为什么没出现或出现异常
- 帮你理解模型切换为何会影响工具面板
- 帮你知道新工具应如何进入系统

## 常见误区

- 以为工具系统只是一个静态数组
- 以为插件工具和内置工具没有本质区别
- 以为模型切换只影响回答风格，不影响工具能力

## 本课结论

工具系统不是静态清单，而是受模型、权限、配置、插件共同作用的动态编排层。
