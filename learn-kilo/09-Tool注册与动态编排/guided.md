# 带读讲义

## 这份带读适合什么时候用

- 你第一次读 `ToolRegistry`
- 你知道有工具系统，但不知道它是怎么拼起来的

## 阅读前准备

第一次阅读只回答一个问题：

“工具到底是怎么进入系统的？”

## 第 1 步：先看 `all()`

只看这里，不要一开始跳到每个工具实现。

把工具分成 3 类：

- 默认内置工具
- 条件启用工具
- 自定义 / 插件工具

### 停下来回答

- `all()` 为什么不是一个固定常量数组？

## 第 2 步：看 `tools(...)`

理解：

- 为什么还要再过滤一次
- 模型为什么会影响工具可见性

### 停下来回答

- 为什么 `apply_patch` 与 `edit/write` 会按模型切换？

## 第 3 步：再看 `fromPlugin`

感受插件工具如何被包进统一结构。

### 停下来回答

- 插件工具为什么需要统一适配层？

## 第 4 步：最后再选一个具体工具

推荐：

- `read.ts`
- `bash.ts`
- `apply_patch.ts`

只看参数、描述、执行入口。

## 第一次阅读先停在哪

你能说清：

- 工具来源
- 工具过滤
- 模型影响
- 插件接入

就可以先停。

## 读完后的最小输出

写一段说明：

“一个工具从定义到进入模型可见范围，中间经历了什么。”
