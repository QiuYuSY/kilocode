# 11 课：SDK 与契约层

## 适合谁

- 已看过客户端连接，想理解共享桥梁的人

## 前置要求

- 完成 10 课

## 预计时长

- 总时长：1.5 到 2.5 小时
- 推荐分 2 次完成

## 你已经学会了什么

- 你已经知道扩展是如何连接 CLI backend 的
- 你已经知道客户端需要共享统一后端能力

## 这节课新增什么

- SDK 的契约角色
- 自动生成的价值
- 多客户端共享接口层

## 完成线

- 最低完成线：知道 `packages/sdk/js` 在做什么
- 标准完成线：能解释为什么 SDK 必须存在
- 深入完成线：能解释为什么它更像契约层而不是工具库

## 难度梯度

1. 观察：认 SDK 入口
2. 追链：认 client 创建过程
3. 解释：认自动生成的价值
4. 思考：比较 SDK 与手写 fetch

## 源码阅读顺序卡

1. `packages/sdk/js/src/client.ts`
2. `packages/sdk/js/src/index.ts`
3. `packages/sdk/js/src/v2/`

## 必须关注的函数/结构

- `createKiloClient`
- `KiloClient`
- `gen/` 与 `v2/` 的角色

## 这课暂时不用深挖

- 所有生成代码的完整细节
- 每个请求类型的全部定义

## 在真实开发中有什么用

- 帮你理解服务端改动为什么经常伴随 SDK 变更
- 帮你知道客户端报错时是否可能是契约不一致
- 帮你为未来新增接口时找到正确流程

## 常见误区

- 把 SDK 当作普通 fetch 包装器
- 以为自动生成只是省事，不是契约保障
- 忽略 SDK 对多客户端一致性的作用

## 本课结论

SDK 在这里不是“图方便”，而是多客户端共享后端契约的重要一层。

## 下一课为什么学它

SDK 看懂后，下一步就该看共享前端和组件层是如何建立起来的。
