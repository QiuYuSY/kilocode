# 14 课：毕业实战与深度复盘

## 适合谁

- 已经完成前面课程，准备把理解变成成果的人

## 前置要求

- 完成 00 到 13 课

## 预计时长

- 总时长：4 到 8 小时
- 推荐分 3 到 5 次完成

## 你已经学会了什么

- 你已经完成了从产品层到运行时层、客户端层、Kilo 增量层的全景学习
- 你已经具备独立讲清核心链路的基础

## 这节课新增什么

- 毕业级输出要求
- 成果分层
- 对整个学习过程的深度复盘

## 完成线

- 最低完成线：完成一份源码讲解
- 标准完成线：完成一次小型真实改动或同等级成果
- 深入完成线：同时完成讲解、改动、复盘三项

## 难度梯度

1. 观察：选题
2. 追链：整理证据链
3. 解释：形成可讲清的成果
4. 思考：对整个学习过程做深度复盘

## 源码阅读顺序卡

按你的选题反向使用前面课程中的阅读顺序卡。

## 必须关注的结构

- 入口文件
- 关键模块
- 数据流
- 设计点
- Kilo / upstream 差异

## 这课暂时不用深挖

- 与你选题无关的其他模块
- 不影响你形成成果的边缘实现

## 在真实开发中有什么用

- 帮你把理解转化为可复用成果
- 帮你检验自己是否真的具备定位问题和讲清问题的能力
- 帮你为之后的真实协作、PR、技术分享打基础

## 常见误区

- 觉得前面都看过了，毕业课就不用做
- 只做描述性总结，不做真正证据链
- 选题过大，最后讲不清也改不动

## 本课结论

真正学会一个工程，不是“看过”，而是“能解释、能输出、能改一点东西”。

## 下一步还能做什么

你可以继续把自己的毕业成果升级成：

- 一篇系统讲解文章
- 一次真实 PR
- 一份自己的项目理解笔记库
