# Learn Kilo 第二轮重构实施任务列表

本轮重构的目标只有一句话:

让学习者先通过课程本体完整掌握 KiloCode 工程，再把源码页当成验证和定位工具，而不是主教材。

## 总体原则

- 先教材，后源码
- 先讲工程，后看实现
- 先给判断，后给证据
- 课程本身要能支撑二开，不依赖读者自行从代码里补课

## 任务 1: 固化本轮改造范围

目标:

- 明确本轮不是新增零散内容，而是升级每节课的讲义结构

完成标准:

- 形成这份实施任务列表
- 后续改造按任务顺序推进

## 任务 2: 升级 01-08 课讲义结构

目标:

- 每节 `02-课程讲义.md` 不再只是解释源码要点
- 改成课程本体可独立教学的结构

统一新增 4 个部分:

- 脱离源码版主讲
- 关键抽象解剖卡
- 常见二开场景
- 源码映射附录

完成标准:

- `01-08` 每节讲义都具备这 4 个部分

## 任务 3: 加强“关键抽象解剖卡”

目标:

- 让学习者不依赖源码也能回答:
  - 它是什么
  - 为什么必须存在
  - 没有它会怎样

覆盖范围:

- Project
- Instance
- Session
- Part
- SessionProcessor
- Agent
- Permission
- ToolRegistry
- ServerManager
- KiloConnectionService
- Kilo 差异层核心包

完成标准:

- 每节核心课都有至少 2 到 4 张抽象解剖卡

## 任务 4: 加强“常见二开场景”

目标:

- 让学习者看到常见需求时，先想到落点和路径，而不是先去翻代码

每节课都要补:

- 本课相关的常见二开需求
- 正确落点
- 主要风险
- 最小验证方式

完成标准:

- `01-08` 每节讲义都有二开场景段落

## 任务 5: 把源码降级为“映射附录”

目标:

- 明确代码页只承担两件事:
  - 验证课程结论
  - 提供实现落点

而不再承担“主教学内容”。

完成标准:

- 每节讲义最后都有“源码映射附录”
- 明确写出先看课程，再去代码验证

## 任务 6: 同步总纲

目标:

- 让总纲和课程结构说明跟新讲义结构保持一致

完成标准:

- `learn-kilo/README.md` 说明每节课新增的 4 个讲义部分

## 任务 7: 自检

目标:

- 确认这轮改造后，课程确实往“独立教材”方向推进

完成标准:

- 抽查 `01-08` 每节讲义是否都具备新结构
- 抽查总纲是否同步
- 输出改造结果摘要
