# 课程优化清单

这份清单不是重做课程方向，而是在当前“先课程本体、再源码验证”的主线上继续提完成率、复用性和交付感。

## 本轮先修

### P0: 可用性

- 修复 `course-site` 初始化报错，确保入口页打开后能完整渲染
- 把 `course-site` 文档链接转成相对路径，避免依赖本机绝对目录
- 持久化当前课节、视图和模块筛选，支持刷新后断点续学
- 统一入口文档中的关键链接格式，减少 README 首屏跳转失效

完成标准:

- 本地直接打开或用静态服务打开 `course-site/index.html` 都能正常渲染
- 主教材、课程讲义、源码入口链接可点击
- 刷新页面后仍保留当前学习位置

## 下一轮增强

### P1: 教学强化

1. 毕业项目增加标准答案对照
   - 每个题目给出范围约束、最低证据链、及格稿结构、常见误判
2. 关键链路补时序图
   - 优先补 `02`、`03`、`04` 三课的主链和交互链
3. 增加不同角色快速路径
   - 至少覆盖新人通读、CLI 二开、扩展端二开、维护者校准 4 条路径
4. 为模块 A/B/C/D 增加闸门
   - 每个模块结束时要求一个固定交付物，而不只是课内勾选

完成标准:

- 学习者能知道“下一步该看什么”和“这一阶段算不算过关”
- 毕业项目不再只靠自我感觉判断是否完成

### P2: 学习体验

1. 加课程搜索和直达
   - 支持按课名、模块、关键对象快速定位
2. 增加深链接
   - 允许直接分享某一课、某个 tab、某个模块筛选状态
3. 增加进度重置和导出
   - 支持清空本地进度，或导出自己的学习记录模板
4. 增加“我现在适合做什么”提示
   - 根据当前进度提示继续主教材、继续分课或开始毕业项目

## 暂不优先

- 全量视频化
- 每节课都做动图
- 把课程入口页重构成完整前端工程

这些方向有价值，但当前阶段不如“标准答案、时序图、角色快路径、模块闸门”更直接提升训练效果。

## 建议执行顺序

1. 先收口入口页和 README 可用性
2. 再补毕业项目标准答案对照
3. 再补关键链路时序图
4. 再加角色快速路径和模块闸门
5. 最后做搜索、深链接、导出这类体验增强
