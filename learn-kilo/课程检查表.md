# 课程检查表

## 通用自检问题

每学完一课，都问自己：

1. 我能不看笔记讲清这课的核心结论吗？
2. 我能画出这课的核心关系图或链路图吗？
3. 我能解释至少一个设计选择的原因吗？
4. 我能说出一个替代设计以及它的代价吗？

如果这 4 个问题里有 2 个答不上来，就先不要进下一课。

## 逐课检查

### 00 课

- 我已经有自己的学习目标和风险清单
- 我知道每一课要怎么学、怎么交付
- 我知道为什么不能直接乱看源码

### 01 课

- 我能把主要包分成核心引擎、客户端、共享层、Kilo 增量
- 我能画出产品关系图
- 我能说清为什么 `packages/opencode` 是第一优先级

### 02 课

- 我能解释 CLI 入口做了哪些全局工作
- 我能说出 `kilo serve` 是怎么进入 `ServeCommand` 的
- 我能解释为什么命令系统要集中注册

### 03 课

- 我能说清 Server 的主要中间件与路由族
- 我能解释 `/event` 的角色
- 我能比较 SSE 与 polling 的差异

### 04 课

- 我能说清 Project、directory、worktree、Instance 的关系
- 我能解释为什么要按目录隔离状态
- 我能说出全局单例在这里为什么不够

### 05 课

- 我知道配置不是只读一个文件
- 我能列出主要配置来源
- 我能解释数据库与文件存储的角色分工

### 06 课

- 我能区分 Session、Message、Part
- 我能解释为什么 Message 要拆 Part
- 我能画出 Session 数据模型图

### 07 课

- 我能追踪一次 tool call 生命周期
- 我能解释 `SessionProcessor` 为什么像事件驱动执行器
- 我能说出 retry、patch、summary 的位置

### 08 课

- 我能区分 `code`、`plan`、`ask`、`debug`
- 我能解释为什么模式差异不只是 prompt
- 我能说明 permission 是运行时安全边界

### 09 课

- 我能解释 `ToolRegistry` 的角色
- 我能追踪一个工具从定义到暴露给模型的过程
- 我能说明为什么工具集需要动态编排

### 10 课

- 我能解释 extension host、webview、CLI backend 三层关系
- 我能说明 `ServerManager` 和 `KiloConnectionService` 的职责
- 我能解释 Agent Manager 为什么能多会话但单后端

### 11 课

- 我能解释 SDK 为什么不是可有可无的工具库
- 我能说清 `createKiloClient` 做了什么
- 我能说明为什么 SDK 更像契约层

### 12 课

- 我能解释 `packages/app`、`packages/kilo-vscode/webview-ui`、`packages/kilo-ui` 的关系
- 我能说明为什么共享前端不等于完全统一 UI
- 我能说清组件层的职责

### 13 课

- 我能解释什么是 Kilo-specific 增量
- 我能说明 `kilocode_change` 的意义
- 我能说清 fork 差异为什么要保持可识别

### 14 课

- 我已经完成至少一种毕业成果
- 我能脱稿讲清一条完整链路
- 我能对自己对整个仓库的理解深度做出复盘

## 毕业检查

学完整套课后，你应该能：

- 解释产品关系
- 解释核心运行链路
- 解释关键抽象
- 解释客户端接入方式
- 解释 Kilo 增量能力
- 做出一份可检验成果
